### Problem: Fullscreen Compose Window Clipping with Sidebar Active

In the current implementation, when the Wingman sidebar is active, fullscreen compose windows are forced to a specific width and right-offset using `calc(100vw - 350px)` and `right: 350px`. 

**Observed Issues:**
1. **Clipping:** Gmail's internal header buttons (Minimize, Maximize, Close) in the compose window are pushed "out of frame" to the right.
2. **Layout Calculation Conflict:** Gmail's internal JavaScript calculates the position of these buttons based on the window width. When we force a width via CSS/JS without adjusting the `left` position or allowing the browser to flow naturally, these elements break.
3. **Inconsistency:** Some parts of the code use `345px` while others use `350px`.

### Proposed Fix

We should use `left: 0 !important` and `right: 350px !important` with `width: auto !important`. This forces the browser to pin the window to both edges (left screen edge and sidebar edge), which is more robust for responsive layout and ensures Gmail's internal elements have the expected container boundaries.

#### 1. Update `client/js/content-compose.js`

Modify the `observeComposeWindows` function (around line 344) to set `left` and `right` instead of `width`:

```javascript
            if (isFullscreen && isSidebarActive) {
                dialog.classList.add('wm-fullscreen-compose');
                dialog.style.setProperty('right', '350px', 'important');
                dialog.style.setProperty('left', '0', 'important');
                dialog.style.setProperty('width', 'auto', 'important');
                dialog.style.setProperty('max-width', 'none', 'important');
            } else {
                dialog.classList.remove('wm-fullscreen-compose');
                dialog.style.removeProperty('right');
                dialog.style.removeProperty('left');
                dialog.style.removeProperty('width');
                dialog.style.removeProperty('max-width');
            }
```

#### 2. Update `client/css/content-gmail-layout.css`

Modify the CSS rules for fullscreen compose (around line 75):

```css
body.wm-sidebar-active div.nH.Hd[role="dialog"][style*="position: fixed"],
body.wm-sidebar-active div.nH.Hd[role="dialog"].wm-fullscreen-compose {
    right: 350px !important;
    left: 0 !important;
    width: auto !important;
    max-width: none !important;
}
```

#### 3. Ensure consistency
Standardize on `350px` for the sidebar width across all layout calculations (check `content-compose.js` line 326 where `345px` is currently used).

```javascript
// Line 327 in content-compose.js
container.style.setProperty('margin-right', '350px', 'important');
```

This change will ensure the compose window "fits" perfectly in the remaining Gmail UI space without clipping internal elements.
